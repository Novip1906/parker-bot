# Parker Bot

Телеграм-бот, который генерирует тексты в стиле заданного телеграм-канала. Для работы используется LLM-провайдер и системный промпт, сформированный на основе реальных публикаций из Telegram-канала.

## Инструкция по использованию и запуску

### 1. Настройка переменных окружения (.env)

Прежде всего необходимо настроить ключи доступа. В корневой директории проекта создайте или отредактируйте файл `.env`:

1. **Регистрация бота:**
   - Перейдите в Telegram к [@BotFather](https://t.me/BotFather).
   - Отправьте команду `/newbot`, задайте имя и юзернейм для вашего бота.
   - Скопируйте полученный HTTP API токен.

2. **Заполнение `.env` файла:**
   Вставьте токен бота, ваш API-ключ от LLM-провайдера и укажите модели. Пример содержимого `.env`:
   
   ```env
   TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
   AI_PROVIDER_API_KEY=ваш_api_ключ_от_провайдера_llm
   AI_PROVIDER_BASE_URL=https://polza.ai/api/v1 # или другой базовый URL
   
   # Модели
   AI_MODEL_NAME=google/gemini-2.5-flash
   MODEL_TRAINING_MODEL_NAME=google/gemini-2.5-pro
   ```

### 2. Подготовка обучающих данных

Для того чтобы бот перенял стиль автора, необходимо предоставить ему массив реальных постов:

1. Сделайте экспорт истории постов (рекомендуется несколько сотен сообщений) из нужного Telegram-канала в формате **JSON**.
2. В результате вы получите файл (обычно `result.json`). Переместите его в директорию `model-training`.
3. Обязательно переименуйте этот файл в `raw_data.json`.

### 3. Очистка данных и генерация промпта

Теперь нужно обработать данные и сформировать системный промпт:

1. Запустите скрипт очистки данных:
   ```bash
   python model-training/clean_data.py
   ```

2. Сразу после этого запустите скрипт генерации промпта (он использует настройки из `.env` для обращения к LLM):
   ```bash
   python model-training/generate_prompt.py
   ```

### 4. Запуск бота

Когда промпт сгенерирован, можно запускать бота:

```bash
python bot/bot.py
```

После этого бот готов к работе. Перейдите в Telegram, найдите вашего бота и отправьте ему сообщение — он должен ответить вам в фирменном стиле автора!
